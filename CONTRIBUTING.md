# Contributing to Worker Orchestrator

Obrigado pelo interesse em contribuir! Este é um projeto focado em demonstrar engenharia de software sólida.

## Como contribuir

1. **Bug Reports & Ideas:** Abra uma Issue detalhando o problema ou sugestão.
2. **Pull Requests:** 
    - Faça um fork do repositório.
    - Crie uma branch para sua feature (`git checkout -b feature/nova-ideia`).
    - Certifique-se de que os testes passam (`pytest tests/`).
    - Envie o PR com uma descrição clara do que foi alterado.

## Requisitos de Qualidade
- Mantenha a tipagem em dia.
- Adicione testes de integração se criar novas funcionalidades de banco.
- Siga as ADRs (Architectural Decision Records) existentes em `docs/adr/`.
